# 30-01 CliRelay SCE 接管

## 1. 概述

当前仓库已经具备记忆系统、SCE 检索引擎和控制台，但此前没有正式的 `.sce/` 工作入口。

本 Spec 的目标是：

1. 为仓库建立 SCE 接管骨架
2. 明确当前项目结构和接管事实
3. 把下一阶段重点问题转成可执行任务入口

## 2. 需求

### 2.1 建立 `.sce/` 项目入口

**验收标准**

1. 仓库根目录存在 `.sce/`
2. `.sce/README.md` 能说明项目入口和阅读路径
3. `.sce/steering/` 至少包含 manifest、核心原则、环境、当前上下文、规则索引

### 2.2 建立当前项目参考 Spec

**验收标准**

1. 存在 `00-00-project-reference`
2. 能快速定位核心代码入口和核心文档入口

### 2.3 建立当前接管 Spec

**验收标准**

1. 存在 `30-01-clirelay-sce-takeover`
2. 能明确当前接管目标和后续任务

### 2.4 与现有仓库文档联动

**验收标准**

1. `.sce` 文档不与当前仓库文档矛盾
2. 至少有一处明确链接到 `SCE_GATEWAY_TASKS.md` 或 `docs/project-status_CN.md`
